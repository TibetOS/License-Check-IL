# License-Check-IL 🚗

בדיקת פרטי כלי רכב לפי מספר לוחית רישוי — רכב פרטי, אופנועים, משאיות, אוטובוסים וצמ"ה — ישירות ממאגרי כלי הרכב של משרד התחבורה ב-[data.gov.il](https://data.gov.il/dataset/private-and-commercial-vehicles).

אפליקציית עמוד יחיד (HTML/CSS/JS בלבד), ללא שרת: השאילתות נשלחות מהדפדפן ישירות ל-API הציבורי של data.gov.il.

## מה מוצג

- פרטי הרישום המלאים מהמאגר הראשי, כולל מפרט דגם (כוח סוס, הנעה, מרכב, מספר מושבים ודלתות, ארץ תוצרת, CO₂, קבוצת אגרה), אבזור נוחות (מיזוג אוויר, גג נפתח, חישוקי סגסוגת), מערכות בטיחות מותקנות, מחיר מחירון ויבואן.
- **הצלבת מספר השלדה מול היצרן** — קידומת ה-VIN (WMI) מפוענחת מקומית (ללא בקשת רשת) ומוצגת כ"יצרן לפי מספר השלדה"; התאמה ליצרן הרשום מסומנת בתג ירוק, ואי-התאמה — סימן אפשרי לרכב "משוכפל" — מוצגת בתג אדום ובאזהרה בולטת בשורת התמצית. ההתאמה נבדקת ברמת קבוצת היצרנים (רכב ממותג-מחדש אינו מסומן בטעות), וקידומת לא מוכרת פשוט אינה מוצגת — לעולם אין התרעה על סמך ספק.
- **היסטוריית רכב** (כיסוי חלקי — בעיקר רכבים חדשים): מד אוץ בטסט האחרון, ממוצע נסועה שנתי מוערך, רישום ראשון, דגלי שינויים (מבנה / גפ"מ / צבע / צמיגים) והחלפות בעלות עם ספירת יד.
- **סיפור הרכב — ציר זמן** — ביוגרפיה כרונולוגית שנבנית מהנתונים הקיימים: עלה לכביש, כל החלפת בעלות (יד 1, יד 2…), קריאות ריקול שנפתחו, הטסט האחרון עם מד האוץ, סמן "היום", ותוקף הרישיון כאירוע עתידי; ברכב מבוטל — אירוע הביטול עם הסיבה. אירועים מצטרפים לציר בזמן אמת ככל שתשובות המאגרים חוזרות.
- **כמה כאלה על הכביש** — ספירת רכבים מאותו דגם (ולפי שנת ייצור) במאגר הפעיל, כאינדיקציה לנפוצוּת/נדירוּת.
- לאופנועים: דרגת הרישיון הנדרשת (משוערת לפי נפח והספק), סיווג EU, משקל, מספר מקומות ודירוג צמיגים.
- לרכב ציבורי: העשרה מצי האוטובוסים — חברה מפעילה, ק"מ מצטבר, סוג ומיגון.
- לכלי צמ"ה: דרגת זיהום אוויר, מסנן חלקיקים ואישור פעילות (חיווי אם הכלי אינו מורשה לפעול).
- קריאות ריקול פתוחות, כולל יבואן, טלפון ואופן תיקון — וכשאין ריקול פתוח, חיווי ירוק מפורש "אין קריאות ריקול פתוחות".
- חיווים נוספים: תו חניה לנכה (כולל "אין תו" מפורש), מערכת בטיחות מזכה בהנחת אגרה, מסנן חלקיקים, חובת עיגון מטען.
- הבחנה עקבית בין "אין" לבין "אין נתונים": מאגרים מוסמכים (ריקולים, תווי חניה) מציגים "אין" מפורש; המאגר החלקי של ההיסטוריה מציג "לא נמצאו נתונים"; כשל רשת לעולם לא מוצג כ"אין".
- **קישור ישיר לתוצאה** — המספר הנבדק משתקף בכתובת (`?plate=1234567`), וכפתור שיתוף בכרטיס משתף את הקישור (Web Share בנייד, העתקה ללוח בדסקטופ). פתיחת קישור כזה מריצה את הבדיקה אוטומטית.
- **כרטיס שיתוף כתמונה** — בדפדפנים עם שיתוף קבצים (וואטסאפ בנייד וכו'), כפתור השיתוף מפיק תמונת תמצית שנוצרת במכשיר: הלוחית הצהובה, שם הדגם, תמונת הדגם (אם נטענה), עובדות מפתח וחותמת תאריך הבדיקה — והקישור החי מצורף לצדה. בלי תמיכת קבצים — משותף הקישור כרגיל.
- **תמונת הדגם — מדויקת לדור** — תמונה מייצגת מוצגת בכרטיס (מסומנת "להמחשה בלבד" — עשויה להציג צבע אחר). כ-230 הדגמים הנפוצים בישראל (כ-63% מהרכבים בכביש) שמורים בריפו עצמו בתיקיית `model-images/`, ולדגמים המובילים גם תמונה נפרדת לכל דור — הבחירה נעשית לפי שנת הייצור, כך שקורולה 2008 מקבלת את הדור הנכון ולא את החדש. התמונות ברישיון חופשי מוויקישיתוף, עם ייחוס מלא לצלם ולרישיון בכיתוב שמתחת לתמונה וב-[ATTRIBUTION.md](model-images/ATTRIBUTION.md), נטענות מקומית בלי לשלוח דבר לשרת חיצוני (ועובדות גם ללא רשת). דגם שאינו במאגר המקומי מחפש תמונה בוויקיפדיה כבעבר: נשלח שם הדגם בלבד, לעולם לא מספר הרישוי; כשאין התאמה ודאית לא מוצגת תמונה כלל.
- **לוגו היצרן** — סמל היצרן מוצג בכותרת הכרטיס. הלוגואים (כ-70 מותגים) שמורים בריפו עצמו בתיקיית `logos/` (מתוך [car-logos-dataset](https://github.com/filippofilip95/car-logos-dataset)) ונטענים מקומית — ללא בקשת רשת חיצונית; יצרן ללא לוגו במאגר פשוט מוצג בלי סמל.
- **שורת כותרת אינפורמטיבית** — גיל הרכב וסוג הבעלות במשפט קצר מתחת לכותרת, והשוואת הנסועה השנתית לממוצע הארצי בהיסטוריה.
- **הסברי "מה זה?" עם מקור** — מונחים מקצועיים בכרטיס (קבוצת זיהום, מדד ירוק, ניקוד בטיחות, קבוצת אגרה, מספר שלדה ועוד) מקבלים כפתור ⓘ שפותח הסבר קצר בשפה פשוטה ושורת מקור הנתון — שקיפות מלאה על מקור כל שדה.
- **הדבקה חכמה** — מדביקים לשדה החיפוש טקסט שלם (מודעת יד 2, הודעת וואטסאפ) ומספר הרישוי מחולץ מתוכו אוטומטית והבדיקה רצה מיד. טלפונים ומחירים אינם נתפסים בטעות; החילוץ כולו מקומי.
- **אגרת רישוי שנתית** — סכום האגרה המשוער מחושב מקומית לפי התעריף הרשמי של משרד התחבורה (רכב פרטי: קבוצת אגרה × שנת ייצור; אופנוע: לפי נפח; משאית: לפי משקל ודלק), מוצג כשורה בכרטיס ובקופסת החידוש, עם ציון גרסת התעריף (04/2026) והבהרה שהסכום המחייב הוא זה שבשירות התשלומים.
- **חידוש רישיון רכב** — כשרישיון הרכב פג (או מסתיים בתוך 30 יום) מוצגת קופסת פעולה עם שלושת שלבי החידוש — תשלום האגרה (קישור לשירות התשלומים הממשלתי, כולל אפשרות טלפונית ופריסה לתשלומים), ביטוח חובה, וטסט — בניסוח מדורג לפי גיל הפקיעה. פקיעה של יותר משנה מופנית לעמוד הרשמי של משרד התחבורה (שם החידוש בדרך כלל דורש טיפול במשרד הרישוי). מוצג רק לרכב פעיל; מידע כללי בלבד.
- **⭐ הרכב שלי** — כפתור בכרטיס שומר רכב אחד (מקומית בלבד, ללא שרת); פאנל במסך הבית מציג אותו עם ספירה לאחור לתוקף הטסט (ירוק / כתום כשנותרו עד 30 יום / אדום כשפג), בדיקה חוזרת בלחיצה והורדת **תזכורת ליומן** (קובץ ‎.ics‎ הנוצר במכשיר, שבועיים לפני תום התוקף); כשהתוקף מסתיים בקרוב או פג מופיע גם קיצור "💳 לתשלום ולחידוש". בדיקה חוזרת של הרכב מרעננת את הנתונים השמורים.
- **סריקת לוחית במצלמה** — כפתור מצלמה בלוחית פותח תצוגה חיה: מכוונים את הלוחית למסגרת, המספר מזוהה אוטומטית (OCR עם Tesseract.js, נטען רק בלחיצה הראשונה) והבדיקה רצה מיד. משוב חי (רצועת קריאה, מסגרת מחליפת צבע), זום בצביטה או בכפתורי ×1/×2/×3 (זום מצלמה מקורי כשנתמך, דיגיטלי אחרת), פנס במכשירים תומכים, וכן **העלאת תמונה** — צילום במצלמת המערכת או בחירה מהגלריה, עם איתור המספר בכל מקום בתמונה. הזיהוי מתבצע כולו במכשיר — אף תמונה אינה נשלחת לשום שרת. הכפתור מוצג רק בדפדפנים עם תמיכת מצלמה, וההקלדה הידנית זמינה תמיד.
- רכב שלא נמצא במאגר הראשי נבדק אוטומטית במאגרים נוספים: לא פעילים, אופנועים, יבוא אישי, רכב ציבורי, רכב כבד (מעל 3.5 טון, כולל רכבי אספנות עם מספר קצר), ביטולים סופיים (כולל ארכיונים עד שנת 2000), וכלי צמ"ה (ציוד מכני הנדסי — מלגזות, מנופים, טרקטורים).

## התקנה כאפליקציה (iOS / Android)

האתר הוא PWA — אפשר להתקין אותו כאפליקציה עם אייקון משלה, ומעטפת האתר נטענת גם ללא רשת (החיפוש עצמו דורש חיבור למאגר הממשלתי):

- **Android (Chrome):** תפריט ⋮ → "הוספה למסך הבית" / "התקנת אפליקציה".
- **iOS (Safari):** כפתור שיתוף → "הוסף למסך הבית".

## הרצה מקומית

אין תהליך build. כל שרת סטטי יעבוד, למשל:

```bash
python3 -m http.server 8000
```

ואז לפתוח את http://localhost:8000 בדפדפן.

(אפשר גם פשוט לפתוח את `index.html` ישירות בדפדפן — ה-API תומך ב-CORS.)

## פריסה ל-GitHub Pages

קיים workflow פריסה אוטומטי (`.github/workflows/deploy-pages.yml`) שמפרסם את האתר בכל push ל-`main`.

הפעלה חד-פעמית: בעמוד ה-repository ב-GitHub, **Settings → Pages**, ותחת **Build and deployment** בחרו Source: **GitHub Actions**. מאותו רגע כל push ל-`main` יפרסם אוטומטית לכתובת `https://<username>.github.io/License-Check-IL/`.

### עמוד Staging (תצוגה מקדימה)

push לענף `staging` פורס את תוכנו לכתובת `https://<username>.github.io/License-Check-IL/staging/` — עמוד חי לבדיקת שינויים לפני מיזוג ל-`main`, בלי לגעת באתר הראשי. כדי להעלות ענף עבודה ל-staging:

```bash
git push origin <branch>:staging --force
```

(ה-Service Worker של כל סביבה מבודד לפי scope, כך ששתי הסביבות אינן מתנגשות במטמון.)
